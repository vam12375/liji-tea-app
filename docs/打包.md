# 安卓 APK 打包说明

## 项目现状

当前项目是 `Expo + Expo Router` 应用，不是已经提交原生工程的
Android 项目。

可从以下配置判断：

- `package.json` 中入口是 `expo-router/entry`
- `app.json` 中存在 Expo 的 `android`、`plugins` 等配置
- 仓库当前没有 `android/` 目录

因此，这个项目**推荐使用 EAS Build 打包 APK**，而不是直接在
Android Studio 中打开原生工程构建。

## 推荐方案：使用 EAS Build 打包 APK

### 1. 补充 Android 包名

当前 `app.json` 里还没有 `android.package`，打包前需要先补上。

示例：

```json
{
  "expo": {
    "android": {
      "package": "com.liji.teaapp"
    }
  }
}
```

建议包名使用你自己的正式命名，例如：

- `com.xxx.lijitea`
- `com.company.lijiteaapp`

### 2. 登录并初始化 EAS

如果还没有登录 Expo 账号，先执行：

```bash
npx eas login
```

初始化 EAS：

```bash
npx eas init
npx eas build:configure
```

执行后，项目里通常会生成 `eas.json`。

### 3. 配置 `eas.json`

为了直接生成 APK，建议增加一个 `preview` 配置：

```json
{
  "build": {
    "preview": {
      "distribution": "internal",
      "environment": "preview",
      "android": {
        "buildType": "apk"
      }
    }
  }
}
```

说明：

- `buildType: "apk"` 表示输出 APK
- `distribution: "internal"` 适合测试安装

### 4. 配置环境变量

项目中 Supabase 使用的是：

- `EXPO_PUBLIC_SUPABASE_URL`
- `EXPO_PUBLIC_SUPABASE_PUBLISHABLE_KEY`

虽然本地开发可以从 `.env` 读取，但云端构建更建议把它们配置到
EAS 环境变量中。

示例命令：

```bash
npx eas env:create --name EXPO_PUBLIC_SUPABASE_URL --value https://你的项目.supabase.co --environment preview --visibility plaintext
npx eas env:create --name EXPO_PUBLIC_SUPABASE_PUBLISHABLE_KEY --value 你的key --environment preview --visibility plaintext
```

说明：

- 这两个值属于前端可见配置，使用 `EXPO_PUBLIC_` 前缀是正确的
- 不建议依赖未提交到仓库的本地 `.env` 去做云打包

### 5. 执行打包

运行：

```bash
npx eas build -p android --profile preview
```

打包完成后，Expo 会返回一个构建链接，进入页面即可下载 `.apk`。

## 本地打包方案

如果你不想走 EAS，也可以本地生成原生 Android 工程后再打包。

### 1. 生成 Android 原生工程

```bash
npx expo prebuild -p android
```

执行后会生成 `android/` 目录。

### 2. 使用 Gradle 打包 Release APK

进入原生目录后执行：

```bash
cd android
.\gradlew assembleRelease
```

### 3. APK 输出位置

通常输出文件在：

```text
android/app/build/outputs/apk/release/app-release.apk
```

## 两种方案怎么选

### 适合选 EAS Build 的情况

- 想最快拿到可安装 APK
- 不想本地折腾 Android Studio、JDK、Gradle
- 想让构建流程更稳定

### 适合选本地打包的情况

- 你已经装好了完整 Android 原生环境
- 你希望完全本地构建
- 你后续会持续修改原生层配置

## 当前项目距离 APK 打包还差什么

按当前仓库状态，至少还缺这两项：

1. `app.json` 中补充 `android.package`
2. 新建 `eas.json` 并加入 APK 构建 profile

如果不补这两项，直接执行 APK 云打包，大概率会卡在配置阶段。

## 建议的最短路径

如果目标只是**尽快产出一个可安装到安卓手机上的 APK**，建议按下面顺序：

1. 在 `app.json` 中增加 `android.package`
2. 初始化 EAS
3. 新建 `eas.json`
4. 配置 Supabase 的 EAS 环境变量
5. 执行 `npx eas build -p android --profile preview`

## 备注

- 如果后续要上架 Google Play，通常应额外配置一个输出 `AAB` 的
  `production` profile
- `APK` 更适合测试分发和手动安装
- `AAB` 更适合正式上架
